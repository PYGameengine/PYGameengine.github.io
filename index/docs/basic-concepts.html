本页面介绍了 PYGameengine 中的基本概念和术语，帮助您更好地理解和使用引擎。在开始开发游戏之前，建议先熟悉这些概念。

引擎核心概念

引擎实例 (Engine Instance)

引擎实例是整个游戏的核心，负责初始化和管理所有子系统。每个游戏通常只有一个引擎实例。

from pygameengine import Engine

# 创建引擎实例
engine = Engine()

# 初始化引擎
engine.initialize()

# 运行游戏主循环
engine.run()

游戏配置 (Game Configuration)

游戏配置包含游戏的各种设置，如窗口大小、标题、帧率限制等。

  • 窗口设置：分辨率、标题、全屏模式
  • 渲染设置：抗锯齿、垂直同步、质量级别
  • 性能设置：帧率限制、线程数

场景管理

场景 (Scene)

场景是游戏中一个独立的环境或关卡，包含一组游戏对象。一个游戏可以有多个场景（如主菜单、游戏关卡、结束画面等）。

  • 包含游戏对象的层次结构
  • 管理场景的加载和卸载
  • 处理场景内的更新和渲染

场景图 (Scene Graph)

场景图是一种树状数据结构，用于组织场景中的游戏对象。每个节点代表一个游戏对象，子节点会继承父节点的变换（位置、旋转、缩放）。

  • 层次化的对象组织
  • 变换继承
  • 高效的对象查找和遍历

实体组件系统

实体 (Entity)

实体是游戏中的一个基本对象，代表场景中的一个事物。实体本身不包含任何功能，而是通过附加组件来获得各种能力。

  • 作为组件的容器
  • 拥有唯一标识符
  • 可以在场景图中定位

组件 (Component)

组件是定义特定功能的代码模块，可以附加到实体上。通过组合不同的组件，可以创建具有各种行为的游戏对象。

  • 渲染组件：负责绘制对象
  • 变换组件：控制对象的位置、旋转、缩放
  • 物理组件：添加碰撞和物理行为
  • 脚本组件：添加自定义逻辑
# 创建实体
player = scene.create_entity("Player")

# 添加组件
player.add_component(TransformComponent(position=(0, 0, 0)))
player.add_component(MeshRendererComponent(mesh="player.obj", material="player_material"))
player.add_component(RigidbodyComponent(mass=1.0))
player.add_component(PlayerControllerComponent())

渲染概念

渲染器 (Renderer)

渲染器负责将游戏世界绘制到屏幕上，处理所有与图形相关的操作。

  • 管理渲染管线
  • 处理相机和视角
  • 优化绘制性能

相机 (Camera)

相机定义了玩家在游戏世界中看到的视角，相当于游戏中的"眼睛"。

  • 控制视野和透视
  • 处理场景的渲染顺序
  • 支持多种投影方式（透视、正交）

网格 (Mesh)

网格是由顶点、边和面组成的3D模型数据，定义了物体的形状。

  • 包含顶点位置、法线、UV坐标等数据
  • 可以从文件加载或程序化生成
  • 支持静态和动态网格

材质 (Material)

材质定义了物体的外观属性，如颜色、纹理、光泽度等。

  • 包含着色器程序和参数
  • 管理纹理和贴图
  • 控制光照交互

光源 (Light)

光源定义了场景中的光照效果，影响物体的亮度和颜色。

  • 方向光：模拟太阳光等远距离光源
  • 点光源：从一个点向各个方向发射光线
  • 聚光灯：有方向性和范围的光源
  • 区域光：从一个区域发射光线

物理概念

物理世界 (Physics World)

物理世界是物理系统的容器，管理所有物理对象和模拟。

  • 管理物理模拟更新
  • 处理碰撞检测
  • 应用物理规则和约束

碰撞体 (Collider)

碰撞体定义了物体的碰撞形状，用于检测碰撞。

  • 盒形碰撞体：矩形或立方体形状
  • 球形碰撞体：圆形或球体形状
  • 胶囊体碰撞体：圆柱体两端加上半球
  • 网格碰撞体：使用复杂的网格形状

刚体 (Rigidbody)

刚体组件使物体受到物理力的影响，如重力、碰撞力等。

  • 控制质量和惯性
  • 处理速度和加速度
  • 支持动力学、运动学和静态模式

输入系统

输入映射 (Input Mapping)

输入映射将原始输入（如键盘按键、鼠标移动）映射到游戏动作（如移动、跳跃、射击）。

  • 配置输入按钮和轴
  • 支持多键绑定
  • 允许玩家自定义控制设置

输入设备 (Input Devices)

输入设备包括键盘、鼠标、游戏手柄等，用于接收玩家的输入。

  • 键盘和鼠标
  • 游戏手柄和控制器
  • 触摸输入（移动设备）

音频系统

音频源 (Audio Source)

音频源组件用于播放声音，可以附加到游戏对象上。

  • 控制音量、音高和循环
  • 支持3D空间音频
  • 管理播放状态（播放、暂停、停止）

音频监听器 (Audio Listener)

音频监听器定义了游戏中的"耳朵"，通常附加到玩家角色或相机上。

  • 接收和处理音频源发出的声音
  • 影响3D空间音频的定位和衰减
  • 每个场景通常只有一个监听器

资源管理

资源 (Resource)

资源是游戏中使用的各种资产，如模型、纹理、音频、脚本等。

  • 可以从文件加载或程序化生成
  • 由资源管理器统一管理
  • 支持异步加载和缓存

资源管理器 (Resource Manager)

资源管理器负责加载、缓存和释放游戏中的各种资源。

  • 统一的资源加载接口
  • 资源缓存和引用计数
  • 资源依赖解析
  • 内存使用优化

脚本系统

脚本组件 (Script Component)

脚本组件允许你使用Python编写自定义逻辑，附加到游戏对象上。

  • 定义游戏对象的行为
  • 处理输入和事件
  • 与其他组件交互
from pygameengine import ScriptComponent

class PlayerController(ScriptComponent):
    def __init__(self):
        super().__init__()
        self.speed = 5.0
    
    def update(self, delta_time):
        # 处理输入
        if self.engine.input.is_key_pressed("W"):
            # 向前移动
            transform = self.get_component("TransformComponent")
            transform.position += (0, 0, -self.speed * delta_time)
        
        # 检测碰撞
        rigidbody = self.get_component("RigidbodyComponent")
        collisions = rigidbody.get_collisions()
        for collision in collisions:
            print(f"碰撞对象: {collision.entity.name}")

游戏循环

游戏循环 (Game Loop)

游戏循环是游戏的核心循环，不断更新游戏状态和渲染画面。

  • 输入处理：收集和处理玩家输入
  • 更新：更新游戏逻辑、物理、AI等
  • 渲染：将游戏世界绘制到屏幕上
  • 等待：控制帧率，避免CPU过载
# 简化的游戏循环伪代码
while running:
    # 处理输入
    process_input()
    
    # 更新游戏状态
    update(delta_time)
    
    # 渲染画面
    render()
    
    # 控制帧率
    wait_frame()
